Manuel utilisateur

Introduction

Ce manuel a pour but de vous guider dans l'utilisation de l'application web Covid-Infograph.

Présentation de l'application

L'application web Covid-Infograph a été développée dans le cadre d'un projet en BUT informatique de l'Université de Toulouse. Voici le dépot GitHub du projet : Repository GitHub

Cette application permet le reportinng d'articles et études scientifiques sur la COVID-19 (Pour plus de détail cliquez ici).

L'application est composée de trois parties :

  • Une partie de visualisation des données sous forme de graphiques. Voir plus
  • Les données brutes que vous pouvez filtrez et télécharger. 1 , 2, 3 , 4
  • Et une partie moteur de recherche qui permet de rechercher parmis tous les titres de la base de données (ici)

Visualisation des données

La partie de visualisation des données est accessible depuis la page d'accueil de l'application.

L'utilisateur voit une multitude de graphiques sur son écran qui sont légendés et qui permettent de visualiser la répartition des données. En haut de cette page, se trouve un champ nommé mois. Ce champ doit être rempli de la manière suivante : MM-YYYY (ex: 01-2021 -> Janvier 2021). Une fois le champ rempli, et l'utilisateur ayant validé sa saisie en cliquant sur le bouton submit, les 2 graphiques suivants s'actualisent avec les données du mois sélectionné :

  • Le graphique nommé "Nombre de publications par concept clés"
  • Le graphique nommé "Nombre de données par revue"

Visualisation des données brutes

Chaque table de la base de données à son onglet dans le menu en haut du site. Une fois une des pages chargés (par exemple la page 1), l'utilisateur peut choisir de filtrer les données affichés dans les champs de recherche. Type de filtre :

  • Données exacte : L'utilisateur doit saisir la valeur exacte de la donnée qu'il recherche. (Souvent sous forme de sélecteurs) Pour les sélecteurs avec beaucoup de valeurs, le choix multiple est possible.
  • Recherche partielle : L'utilisateur peut saisir une partie de la donnée qu'il recherche. (Pour les champs textuels)
  • Recherche par intervalle : L'utilisateur peut saisir un intervalle de valeurs pour la donnée qu'il recherche. (Tous entêtes contenant 2 champs)

    • Le tri

    Toutes les colonnes sauf les colonnes interventions et le sélecteur sont triables. Pour trier une colonne, il suffit de cliquer sur le nom de la colonne. Le tri est effectué par ordre croissant par défaut. Pour trier par ordre décroissant, il suffit de cliquer une seconde fois sur le nom de la colonne. Pour un tri multicolonne (tri sur plusieurs colonnes), il suffit de cliquer sur le nom de la colonne supplémentaire en gardant la touche Ctrl ou Cmd enfoncée.

    Le téléchargement

    L'utilisateur à le choix entre plusieurs formats de téléchargement :

    • CSV : Format de fichier texte qui permet de stocker des données tabulaires.
    • JSON : Format de fichier texte qui permet de stocker des données structurées.
    • HTML : Format de fichier texte qui permet d'afficher les données dans un navigateur web sous forme de tableau.
    Attention, le téléchargement ne prend en compte que les lignes sélectionnées (càd première colonne cochée).

    Moteur de recherche

    Le moteur de recherche est accessible depuis le menu en haut du site. L'utilisateur peut saisir une partie du titre de l'article qu'il recherche. Une fois la saisie effectuée, l'utilisateur valide la saisie en appyuant sur la touche Entrée. En dessous du champ le nombre de résultats est affiché. Une fois la recherche effectuée, l'utilisateur voit une liste de résultats avec le titre, la date, l'abstract et un lien vers l'article.

    Ajouter son excel dans la base de données

    (Permet aussi de lancer le site en local)

    Pour ajouter son excel dans la base de données, il faut d'abord cloné le dépot GitHub du projet ou le télécharger en zip et le dézipper.

    Une fois le dépot cloné ou dézippé, il faut s'assurer d'avoir python (version 3.10 mininum) installé sur sa machine. Et avoir la bibliothèque pipenv installée.

    Pour installer pipenv, il faut ouvrir un terminal et taper la commande suivante : 

                    
                    pip install pipenv

    Une fois pipenv installé, il faut se placer dans la racine du projet (Dossier contenant manage.py) et lancer la commande suivante qui installe les dépendances du projet : 

                    
                    python -m pipenv sync

    Une fois les dépendances installées, il faut rentrer dans l'environnement virtuel du projet avec la commande suivante : 

                    
                    python -m pipenv shell

    Une fois cela fait il faut mettre le excel dans le dossier excel du projet ('/covid_infograph/files/excel'). Dès que le fichier se trouve dans le dossier vous pouvez lancer la commande suivante : 

                    
                    python .\covid_infograph\files\database_add.py [nom du fichier excel]

    Une fois la commande lancée, le fichier excel est ajouté à la base de données.

    Quelques commandes utiles

    Pour quitter l'environnement virtuel du projet, il faut taper la commande suivante : 

                    
                    exit

    Pour lancer le serveur de développement, il faut se placer dans la racine du projet et taper la commande suivante : 

                    
                    python manage.py runserver

    Pour vider la base de données, il faut taper la commande suivante : 

                    
                    python .\covid_infograph\files\database_add.py clear
    Pour plus d'informations, envoyez nous un mail ou poser votre question dans un des issues du projet sur GitHub.
